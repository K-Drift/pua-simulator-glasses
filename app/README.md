# 牛马预备役

基于初级 PRD 实现的移动端网页 App：把 AI 眼镜采集到的行为画面同步到手机端，以马云单老板视角识别状态、触发大群反馈，并沉淀为部门大群、评定记录和 KPI 待办。

## 本地启动

```bash
npm install
npm run dev
```

默认访问：

```text
http://127.0.0.1:5173/
```

## 核心链路

- 顶部：左上角“跳槽”可打开公司/老板面板，当前仅马云可确认并试听 `/public/voices/mayun.mp3`，其他老板显示音色待解锁。
- 头像：马云像素头像资源位于 `/public/ma-avatar.svg`，首页顶部、跳槽面板和部门大群复用同一套视觉。
- 入口：非 demo 首次进入不再上传简历，直接设定当前 KPI、验收项，并进入马云拷打链路。
- 跳槽：面板以 2×2 竖版大厂/总裁像素图呈现，只有马云可确认，其它总裁卡片保持待解锁。
- 首页：单屏展示当前状态、状态匹配像素图、左下角眼镜实时采集画面、右下角眼镜内展示内容，并保留同步/通报动作。
- 报告总结：部门大群采用两条消息结构，先展示 mock 文案，再展示接口/兜底图片，随后展示 HR、部门 Leader 固定回复、截图提醒和老板“有人截图了！”。
- 评定记录：仅展示一句话总结评定、牛马值/抗压力进度条，以及完成汇报、被 PUA、触发金句、消耗总裁 Token。
- KPI：仅保留当前 KPI todo list 和“和老板聊聊”按钮，点击后进入等待更新态，并尝试调用 `/api/agent/kpi` 获取 agent 生成的 KPI。
- 设置：切换图片来源模式（默认跟随场景，也可固定 category）、采样间隔、自动保存策略，并可重新进入 KPI 设定。

## PRD 覆盖点

- 被动识别：看电脑、看手机、吃饭、摸鱼、和朋友聊天、写东西、一个人默认，以及路演要求的发呆、低头睡觉、看手机、用电脑网购演示脚本。
- 自动触发：监工模式按采样间隔轮询识别结果，触发老板切入、证据截图、部门通报和评定记录。
- 主动触发：首页可直接同步眼镜画面并触发部门通报。
- KPI 跟进：支持 todo 勾选更新进度，点击“和老板聊聊”后等待 agent 接口生成新 KPI。
- 后端对接：Vite 代理将 `/api` 转发到 `http://www.yhaox.top:18091`，当前使用 `GET /images?category=...`、`POST /images/clear`，并在清空失败时用 `DELETE /images/clear?category=...` 兜底。

## 演示入口

```text
/?demo=1
/?demo=1&seed=1&tab=group
/?demo=1&seed=1&tab=group&view=reviews
/?demo=1&tab=okr
```

这些入口会预置演示数据，方便直接查看首页、部门大群、评定记录和 KPI 流程。
